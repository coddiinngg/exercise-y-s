# App Improvement Plan

## 총평
현재 앱은 시각 톤, 정보 구조, 결제 흐름의 기본 뼈대는 잘 잡혀 있습니다.  
다만 실사용 제품 기준으로는 상태관리, 성능, 접근성, 콘텐츠 신뢰도, 결제 이후 사용자 여정 고도화가 필요합니다.

## 우선순위 계획

### 1. Core 안정화 (P0, 1-2일)
1. 탭 전환, 스크롤, 카드 렌더링 성능 점검 (`60fps`, jank 여부)
2. 전역 상태를 명시적으로 정리 (`isPremium`, streak, day progress)
3. 더미 결제 로직을 실제 상태 저장 가능한 구조로 분리 (service layer)
4. 공통 spacing, typography, token 정리로 UI 일관성 고정

### 2. 실사용 기능 완성 (P0, 2-4일)
1. Day 1-30 진행 상태 저장 (완료, 현재 day, 연속일)
2. Streak 계산 로직 구현 (오늘 완료/미완료에 따른 증감)
3. Calendar 실제 데이터 연동 (완료일 하이라이트 자동 반영)
4. Plan 시작 버튼 동작 구현 (실제 루틴 화면 또는 상세 화면 연결)

### 3. 콘텐츠 신뢰도 개선 (P1, 1-2일)
1. 논문 카드 링크를 Scholar 검색 링크에서 실제 논문 URL/DOI로 교체
2. 논문 메타데이터 표준화 (저널, 연도, 근거 레벨)
3. 외부 링크 열기 실패 UX 개선 (재시도, 복사 버튼)

### 4. UX 디테일 고도화 (P1, 2-3일)
1. 애니메이션 규칙 통일 (`120-160ms`, 탭 전환만 최소 적용)
2. 로딩, 빈 상태, 오류 상태 화면 추가
3. 결제 성공 후 안내 플로우 개선 (Toast + 자동 이동 + 첫 액션 유도)
4. 모바일 소형 화면에서 카드 높이/타이포 responsive 튜닝

### 5. 품질/릴리즈 준비 (P0, 2-3일)
1. 위젯 테스트 핵심 6개 작성 (결제 잠금, day unlock, streak 표시, 탭 전환 등)
2. Golden test 2-3개로 UI 회귀 방지
3. Crash/Analytics 이벤트 설계 (paywall open, purchase success, day start)
4. 배포 타깃별 체크리스트 작성 (web/mobile)

## 권장 실행 순서
1. `P0` 우선: 진행 상태 저장 + streak 계산 로직 구현
2. 다음: 논문 링크를 실제 DOI 기반으로 교체
3. 마지막: 테스트/분석 이벤트 추가 후 릴리즈 준비
